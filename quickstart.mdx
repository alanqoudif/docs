## The short version

Every Mibyan project is a conversation with a `projectId`. Inside the conversation, the user writes a request, Atlas decides which agent or tool fits best, and the result appears as an Artifact in the preview panel.

## Usage flow

<Steps>
<Step title="Create an account or sign in">
The app supports Supabase Auth with email and password, OAuth through Lovable Cloud Auth, and phone-number OTP when the AutomaAPI keys are enabled.
</Step>

<Step title="Start from the Composer">
From the home page, write your first request, such as: "Write a pitch deck for a delivery company in Oman" or "Build an Arabic landing page for a real estate service."
</Step>

<Step title="Follow the agent and result">
The platform opens `/chat/$projectId` and starts SSE events from `/api/public/agent-run`. You will see progress messages, tool calls, and then an Artifact in the side panel.
</Step>

<Step title="Edit or export">
You can request changes to the same output. Editing tools read the current Artifact, create a new version, and save versions in `artifact_versions`.
</Step>
</Steps>

## User settings

From `/settings`, the user can edit their name, phone number, and preferences. The `profiles.preferences` field stores custom instructions that can be passed into the agent context.

<Tip>
For best results, specify the audience, desired format, number of pages or slides, tone, and any business or local constraints that should be respected.
</Tip>
