## Overview

Export in Mibyan is split across multiple paths because each Artifact type has different requirements. Documents, spreadsheets, and presentations do not all use the same code.

## Paths

<AccordionGroup>
<Accordion title="DOCX">
Documents usually start as Markdown. Routes such as `file-artifact` and `restyle-docx` handle Office file conversion or formatting. Email and document templates rely on libraries such as `docx`.
</Accordion>

<Accordion title="XLSX">
Spreadsheets are created as structured data through `create_sheet`, then can be exported using SheetJS `xlsx`.
</Accordion>

<Accordion title="PPTX">
Presentations use more than one path: `create_slides` to produce the presentation structure, `generate-deck` to create a richer deck, and `export-deck-pptx` for export. There is also a dependency on `pptxgenjs`.
</Accordion>

<Accordion title="HTML / Website">
HTML pages are previewed directly, and multi-page websites are published through the `/sites/{slug}` route.
</Accordion>
</AccordionGroup>

## Important limits

<Warning>
Matching the visual HTML preview to the final PPTX file is not currently guaranteed 100%. The internal plan in `.lovable/plan.md` points to future work on snapshot export and presets.
</Warning>

## User tips

- Ask for the format explicitly: DOCX, PPTX, XLSX, HTML, or a multi-page website.
- For presentations, specify the number of slides and audience type.
- For long documents, specify the word or page count and required sections.
- For spreadsheets, specify the columns and row count if they matter.
