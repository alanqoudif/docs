## What is an Artifact?

An Artifact is an output saved inside the user's project. It appears in the preview panel and can be edited or exported. Each Artifact is linked to a conversation and user, and its versions are stored in the `artifact_versions` table.

## Core types

<CardGroup cols={2}>
<Card title="Document" icon="file-lines">
Markdown content for reports, plans, articles, and contracts. Rendered through `DocArtifact`.
</Card>

<Card title="HTML" icon="browser">
A full HTML page with inline CSS and JS. Rendered through `HtmlArtifact`.
</Card>

<Card title="Sheet" icon="table">
Spreadsheet data exportable as XLSX. Created through `create_sheet`.
</Card>

<Card title="Slides" icon="presentation">
A slide structure or deck pipeline. Previewed through presentation components and exported through PPTX routes.
</Card>

<Card title="Website" icon="globe">
A multi-page website published at `/sites/{slug}` and stored in `websites` and `website_pages`.
</Card>
</CardGroup>

## Artifact lifecycle

<Steps>
<Step title="Create">
The agent calls a tool such as `create_doc` or `create_website`.
</Step>

<Step title="Store">
The server uses the Supabase admin client to store the output in `artifacts`, linked to `conversationId` and `userId`.
</Step>

<Step title="Preview">
The frontend receives a `tool_result` event and opens the output in `ArtifactCanvas`.
</Step>

<Step title="Edit">
When editing, the tool reads the current version through `read_artifact` or directly from the database, then creates a new version.
</Step>
</Steps>

## Where is the code?

- Preview components: `src/components/mubayyan/*Artifact.tsx`.
- Creation tools: `src/server/agent-tools.server.ts`.
- Frontend stores: `src/lib/artifactsStore.ts` and `src/lib/conversationsStore.ts`.
- Supabase types: `src/integrations/supabase/types.ts`.
