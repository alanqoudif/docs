## Current memory

Mibyan's current context is based on the conversation, messages, and stored Artifacts. This means the agent can work with what was created inside the current project and read an existing Artifact before editing it.

There is currently no guarantee that all of a user's conversations are automatically merged into a general long-term memory for every request. Some interfaces and components mention memory, but it should be documented as an in-development capability unless it is connected to a complete backend.

## Current Agent Swarm

The product concept is a team of agents working around a single request. The current implementation is closer to:

- Analyzing request intent.
- Choosing a suitable agent or persona.
- Running specific tools in a sequential loop.
- Showing agent work status visually.

<Info>
This does not mean every request actually runs 100 independent agents in the background. The phrase "up to 100 agents" appears in product copy as an ambition and visual experience more than as a fixed API contract.
</Info>

## How should this be documented for users?

Use precise wording:

- Correct: "Mibyan routes the request to the right agent and uses multiple tools to create the output."
- Correct: "Current context includes the conversation and its related outputs."
- Avoid: "All your data is saved as long-term memory across all projects" unless it has actually been wired up.
- Avoid: "100 agents always work in parallel" unless that becomes part of the implementation.
