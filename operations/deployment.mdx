## Deployment layers

- The TanStack Start app is deployed as a Cloudflare Worker through `wrangler.jsonc`.
- Database, authentication, and storage run on Supabase.
- The `generate-deck` and `export-deck-pptx` functions are deployed as Supabase Edge Functions.
- The external docs site is deployed through Mintlify from the `docs` directory.

## Environment variables

<AccordionGroup>
<Accordion title="Supabase">
`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `SUPABASE_URL`, `SUPABASE_PUBLISHABLE_KEY`, and `SUPABASE_SERVICE_ROLE_KEY`. The last key is highly sensitive and must not be used in the frontend.
</Accordion>

<Accordion title="AI">
`OPENROUTER_API_KEY` is required for most generation and agent paths. `LOVABLE_API_KEY` is used in the AI proxy, email, and some export routes.
</Accordion>

<Accordion title="Skills">
`SKILLS_ADMIN_TOKEN` protects the skill import route.
</Accordion>

<Accordion title="Phone OTP">
`AUTOMAPI_INSTANCE_ID` and `AUTOMAPI_ACCESS_TOKEN` enable sending OTP codes over SMS.
</Accordion>

<Accordion title="Payments">
`VITE_PAYMENTS_CLIENT_TOKEN` is used by the client depending on the development or production environment.
</Accordion>
</AccordionGroup>

## Supabase

Run migrations from `supabase/migrations`. When deploying Edge Functions, configure their secrets separately from Cloudflare secrets, especially:

- `OPENROUTER_API_KEY`
- `LOVABLE_API_KEY`
- The Supabase URL/keys required by the function

## Cloudflare

`wrangler.jsonc` configures the Worker to run the TanStack Start entry with `nodejs_compat`. Make sure server-side secrets are present in the Cloudflare environment and not in public files.

## Monitoring and diagnostics

- PostHog is enabled in the frontend through `src/lib/posthog.tsx`.
- AI logs can be read from `ai_usage_log`.
- Email logs live in the email tables.
- Any `/agent-run` failure appears as an `error` SSE event or as an HTTP status before the stream starts.

## Pre-deploy checklist

- `pnpm lint`
- `pnpm build`
- Verify Supabase migrations.
- Verify Edge Function secrets.
- Try creating a document, spreadsheet, HTML page, and Salem presentation.
- Test sign-in and OTP if its keys are enabled.
