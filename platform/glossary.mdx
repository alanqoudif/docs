## Core names

<CardGroup cols={2}>
<Card title="Mibyan" icon="sparkles">

</Card>

<Card title="Atlas Swarm Core One" icon="code-branch">

<Card title="Atlas" icon="globe">
The main coordinator name used in docs and marketing. It receives the request, understands it, and chooses the right path.
</Card>

<Card title="master" icon="gear">
The internal ID for the main coordinator in code, especially in `src/data/mubayyan.ts`. When reading the code, treat `master` as Atlas.

## Documentation naming rule

Use "Mibyan" when discussing the product, "Atlas Swarm Core One" when discussing the platform or repository, and "Atlas" when discussing the coordinating agent. If `master` appears in the API, database, or code, it is Atlas's programmatic name, not an additional agent.

## Product terms

- `projectId`: the current conversation or workspace ID. It appears in the `/chat/$projectId` route.
- Artifact: a previewable or editable output, such as a document, HTML page, spreadsheet, presentation, or website.
- Agent run: an agent execution cycle through `/api/public/agent-run`, including tool selection, tool execution, and events sent to the frontend.
- Tool call: a function call inside the agent loop, such as `create_doc` or `create_website`.
- Skill: instructions that can be injected into chat context, stored in the `skills` table and importable through the API.
- Edge function: a Supabase function separate from the TanStack app, such as `generate-deck` and `export-deck-pptx`.
