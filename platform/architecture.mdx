## Overview

`atlas-swarm-core-1` is a TanStack Start app built with React 19, TanStack Router, and Vite. It serves both the frontend and API routes from the same project, and is deployed as a Cloudflare Worker. It relies on Supabase for authentication, data, and storage; OpenRouter for model execution; and Supabase Edge Functions for the presentation pipeline.

```text
Browser
  -> TanStack Start route /chat/$projectId
  -> POST /api/public/agent-run (SSE)
  -> Atlas classifier + tool loop
  -> OpenRouter generation
  -> Supabase conversations/messages/artifacts
  -> ArtifactCanvas preview
```

## Runtime components

<AccordionGroup>
<Accordion title="Frontend and routing">
Page files and API routes live in `src/routes`. The main product route is `/chat/$projectId`, and the root app is in `src/routes/__root.tsx`.
</Accordion>

<Accordion title="Authentication">
Supabase Auth is the foundation. The server verifies Bearer JWTs through `src/server/requireAuth.server.ts`. There is also a phone OTP route and a desktop handoff route for linking with the desktop app.
</Accordion>

<Accordion title="Agent loop">
`/api/public/agent-run` opens an event stream. The agent reads messages and Artifacts, chooses a tool from `src/server/agent-tools.server.ts`, then sends progress and result events to the frontend.
</Accordion>

<Accordion title="Generation">
Most generation goes through OpenRouter. The default model in agent tools is `google/gemini-3.1-flash-lite-preview`. Separate generation routes also exist, such as `doc`, `html`, `sheet`, and `file-artifact`.
</Accordion>

<Accordion title="Presentations">
Salem uses two paths: the `create_slides` tool inside the agent loop, and a separate production pipeline in the Supabase Edge Function `generate-deck`, which emits SSE events such as `brief`, `identity`, `storyboard`, `slide_html`, and `ready`.
</Accordion>
</AccordionGroup>

## Project structure

```text
atlas-swarm-core-1/
  src/routes/                 app pages and API endpoints
  src/components/mubayyan/    chat UI and Artifacts
  src/components/docs/        Arabic in-app documentation
  src/server/                 agent tools, skills, parsers, generators
  src/lib/                    frontend clients, email, stores, auth helpers
  src/data/mubayyan.ts        agent definitions and output types
  supabase/migrations/        tables, policies, and functions
  supabase/functions/         generate-deck and export-deck-pptx
```

## Current feature reality

<Warning>
The current implementation chooses an agent and tools sequentially inside a single request. The Agent Swarm UI presents the idea of teamwork, but not every marketing promise, such as running 100 independent agents in parallel or providing global memory across all projects, is fully implemented in the backend.
</Warning>
