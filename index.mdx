Mibyan is Nuqta's Arabic-first multi-agent platform. Internally, the codebase is named `atlas-swarm-core-1`; in the product it appears as Mibyan, and the main coordinator is marketed as Atlas while some files still call it `master`.

These docs describe the platform as it is actually built: a TanStack Start app deployed on Cloudflare, using Supabase for authentication and data, OpenRouter for generation, and Supabase Edge Functions for the deck production pipeline.

## What does the platform do?

Mibyan receives the user's request in a chat interface, analyzes it, chooses the right agent, and creates an Artifact that can be previewed and exported. The core output types are Markdown documents, HTML pages, spreadsheets, presentations, and multi-page websites.

## Sources of truth

These docs were derived from the app files themselves:

- `atlas-swarm-core-1/src/data/mubayyan.ts` for agent definitions.
- `atlas-swarm-core-1/src/server/agent-tools.server.ts` for agent tools.
- `atlas-swarm-core-1/src/routes/api/public/*` for API endpoints.
- `atlas-swarm-core-1/supabase/migrations/*` and `src/integrations/supabase/types.ts` for data.
- `atlas-swarm-core-1/src/components/docs/*` for the Arabic in-app documentation.

<Warning>
Some in-app marketing pages describe capabilities broader than the current implementation, such as general long-term memory or actually running 100 agents in parallel. These docs distinguish what exists now from the roadmap.
</Warning>
