## Source

Tool definitions and implementations live in `atlas-swarm-core-1/src/server/agent-tools.server.ts`. Each tool runs inside a context scoped by `userId` and `conversationId`, and uses the Supabase admin client together with conversation ownership checks in the agent route.

## Tools

<AccordionGroup>
<Accordion title="create_doc">
Creates a new Markdown document. Suitable for reports, plans, articles, contracts, and feasibility studies. Accepts `title`, `prompt`, `audience`, `tone`, `length`, `length_words`, `target_sections`, `sections`, and `must_include`.
</Accordion>

<Accordion title="update_doc">
Updates an existing document through `artifactId` and `instruction`. The correct Artifact must be read or identified before editing.
</Accordion>

<Accordion title="create_html">
Creates a full HTML page that includes `<!DOCTYPE html>`, CSS inside `<style>`, and JS inside `<script>`. Suitable for landing pages and small single-page websites.
</Accordion>

<Accordion title="update_html">
Regenerates existing HTML based on editing instructions.
</Accordion>

<Accordion title="create_sheet">
Creates a spreadsheet. Suitable for budgets, KPIs, comparisons, and lists. Accepts `columns` and `min_rows` when needed.
</Accordion>

<Accordion title="create_slides">
Creates a presentation as a JSON structure. For more advanced decks, see the `generate-deck` path.
</Accordion>

<Accordion title="read_artifact">
Reads the content of an existing Artifact. Use it before editing a previous output or creating a new output that depends on it.
</Accordion>

<Accordion title="create_website">
Creates a multi-page website and automatically publishes it at `/sites/{slug}`. Stores pages in `websites` and `website_pages`.
</Accordion>
</AccordionGroup>

## Tool events

Tools return events to the agent stream:

```ts
type ToolEvent =
  | { type: "tool_call"; name: string; args: unknown; callId: string }
  | { type: "tool_result"; callId: string; ok: boolean; summary: string; artifactId?: string }
  | { type: "progress"; stage: string; current?: number; total?: number };
```

## Add a new tool

1. Add a new schema to `TOOL_SCHEMAS`.
2. Add an executor inside `executeTool`.
3. Always bind the tool to `userId` and `conversationId`.
4. Emit `tool_call`, `progress`, and `tool_result` so the frontend stays synchronized.
5. Update this page and the API reference if the tool becomes part of the public contract.
