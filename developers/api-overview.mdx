## General status

Mibyan has a real internal API used by the app, but it is not yet a stable public SDK. The `/api-docs` page inside the app still shows that the public API is coming soon. Treat this page as a reference for internal developers or approved integrations.

## Authentication

Most routes use:

```http
Authorization: Bearer <supabase_access_token>
Content-Type: application/json
```

Routes that serve external applications, such as `/api/public/ai/models`, add CORS, but that does not mean every route is public without authentication.

## Core agent API

### `POST /api/public/agent-run`

Runs the agent loop and streams Server-Sent Events.

Request:

```json
{
  "conversationId": "uuid",
  "userMessage": "Write a report...",
  "excludeTools": ["create_slides"]
}
```

Core stream events:

- `agent_assigned`
- `requirements`
- `thinking`
- `progress`
- `tool_call`
- `tool_result`
- `assistant_text`
- `done`
- `error`

### `POST /api/public/classify`

Classifies text as `chat` or `task`, and suggests an agent and clarification questions.

```json
{
  "text": "Create a startup pitch deck"
}
```

Response:

```json
{
  "kind": "task",
  "agent": "salem",
  "title": "Startup pitch deck",
  "questions": []
}
```

## Generation endpoints

The project includes independent generation routes, including:

- `POST /api/public/doc`
- `POST /api/public/html`
- `POST /api/public/sheet`
- `POST /api/public/file-artifact`
- `POST /api/public/restyle-docx`

Use `/agent-run` as the main path for the product experience, and use these routes when you need specialized generation outside the agent loop.

## Skills API

- `GET /api/public/skills`: lists enabled skills.
- `POST /api/public/skills`: chooses a suitable skill for a given text.
- `POST /api/public/skills/import`: admin import protected by `SKILLS_ADMIN_TOKEN`.

## Desktop and AI proxy

- `GET /api/public/ai/models`: lists the allowed models.
- `POST /api/public/ai/chat`: proxy to the Lovable AI gateway.
- `POST /api/public/ai/openrouter/chat`: proxy to OpenRouter.
- `GET /api/public/ai/usage`: usage and cost from `ai_usage_log`.
- `POST /api/public/desktop-handoff/redeem`: redeems a desktop linking code.

## Edge functions

- `generate-deck`: the deck production pipeline, streaming multiple events.
- `export-deck-pptx`: PPTX export.

These are not TanStack routes; they are deployed and configured with secrets through Supabase Edge Functions.
