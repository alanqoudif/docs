## كيف يختار مبيان الوكيل؟

الواجهة ترسل رسالة المستخدم إلى `/api/public/agent-run`. أطلس يحلل النية، يقرأ سياق المحادثة والـ artifacts، ثم يختار أداة أو مسار توليد مناسب. في الكود تظهر الوكلاء في `src/data/mubayyan.ts`، وتظهر شخصيات التنفيذ داخل `agent-run.ts`.

## أنماط المخرجات

الوكلاء لا يملكون مخارج منعزلة بالكامل؛ هم يوجهون نفس أدوات المنصة:

- مستندات Markdown عبر `create_doc`.
- صفحات HTML عبر `create_html`.
- جداول XLSX عبر `create_sheet`.
- عروض JSON/Deck عبر `create_slides` أو `generate-deck`.
- مواقع متعددة الصفحات عبر `create_website`.
- تعديلات عبر `update_doc` و`update_html` بعد قراءة الـ Artifact الحالي.
