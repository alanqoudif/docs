## Salem's role

Salem is the presentation specialist. It is used for requests such as pitch decks, investor presentations, company decks, and visual storytelling sequences.

## Available paths

<AccordionGroup>
<Accordion title="agent-run path">
Inside the agent loop, the `create_slides` tool produces a JSON presentation structure: title, slides, bullet points, and notes. This path is fast and suitable for previewing and editing inside the conversation.
</Accordion>

<Accordion title="generate-deck path">
The Supabase Edge Function `generate-deck` is a broader production pipeline. It streams events such as `status`, `brief`, `identity`, `storyboard`, `slide_html`, `ready`, and `error`. This path is closer to producing a full deck with identity, story, and slide HTML.
</Accordion>

<Accordion title="PPTX export">
The `export-deck-pptx` function exists, along with other routes inside `file-artifact`. The internal plan notes that matching the HTML preview to PPTX export is still an improvement area.
</Accordion>
</AccordionGroup>

## Best prompt for Salem

Write the request with:

- Audience: investors, management, customers, or an internal team.
- Number of slides.
- Goal of the presentation.
- Visual style or brand direction.
- Required sections, such as problem, solution, market, financial model, team, and plan.

```text
Create a 12-slide Arabic pitch deck for a real estate SaaS company in Saudi Arabia, aimed at seed investors, with a premium minimal style, traction, financial model, and a clear ask.
```

## Known limits

PPTX export may not always match the visual preview perfectly. For high-stakes final decks, review the exported file and adjust the template or identity through the prompt when needed.
