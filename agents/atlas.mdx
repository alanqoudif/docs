## من هو أطلس؟

أطلس هو الاسم المنتج للمنسق الرئيسي. في `src/data/mubayyan.ts` معرفه الداخلي هو `master` واسمه العربي في بعض الواجهات "ماستر". دوره ليس إنتاج كل شيء بنفسه، بل فهم الطلب وتوجيهه إلى الأداة أو الوكيل الأنسب.

## ماذا يفعل؟

- يقرأ رسالة المستخدم وسياق المحادثة.
- يقرر هل الطلب دردشة، مستند، عرض، موقع، صفحة HTML، أو جدول.
- يستدعي أدوات مثل `create_doc`, `create_html`, `create_sheet`, أو `create_website`.
- يرسل أحداث تقدم للواجهة عبر SSE.
- يحفظ الناتج في `artifacts` ويعرضه في `ArtifactCanvas`.

## دورة التنفيذ

```text
user message
  -> agent-run
  -> classify / persona
  -> tool call
  -> OpenRouter generation
  -> Supabase artifact
  -> streamed result to UI
```

## حدود التنفيذ الحالي

<Info>
أطلس يقود حلقة أدوات بحد أقصى لعدد من التكرارات داخل request واحد. عبارة "Agent Swarm حتى 100 وكيل" موجودة في بعض نصوص التسويق، لكنها لا تعني حالياً 100 عملية مستقلة متوازية في الخلفية لكل طلب.
</Info>

## مثال

طلب مثل:

```text
اكتب دراسة جدوى مفصلة لمقهى مختص في مسقط مع جدول تكاليف وخطة افتتاح.
```

قد يوجهه أطلس إلى سارة لإنشاء مستند، ويضيف تعليمات تخص ناصر للجانب الاستراتيجي وميرا للجداول. التنفيذ العملي يتم عبر `create_doc` وربما `create_sheet` إذا طلب المستخدم جدولاً منفصلاً.
