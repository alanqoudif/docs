## Who is Atlas?

Atlas is the product name for the main coordinator. In `src/data/mubayyan.ts`, its internal ID is `master`, and some UI surfaces use the Arabic name "Master". Its role is not to produce everything by itself, but to understand the request and route it to the best tool or agent.

## What does it do?

- Reads the user message and conversation context.
- Decides whether the request is chat, a document, a presentation, a website, an HTML page, or a spreadsheet.
- Calls tools such as `create_doc`, `create_html`, `create_sheet`, or `create_website`.
- Sends progress events to the frontend over SSE.
- Saves the output in `artifacts` and displays it in `ArtifactCanvas`.

## Execution flow

```text
user message
  -> agent-run
  -> classify / persona
  -> tool call
  -> OpenRouter generation
  -> Supabase artifact
  -> streamed result to UI
```

## Current implementation limits

<Info>
Atlas drives a bounded tool loop inside a single request. The phrase "Agent Swarm up to 100 agents" appears in some marketing copy, but it does not currently mean 100 independent parallel background processes for every request.
</Info>

## Example

A request like:

```text
Write a detailed feasibility study for a specialty cafe in Muscat with a cost table and opening plan.
```

Atlas may route it to Sara to create a document, adding Nasser-style instructions for the strategic side and Mira-style instructions for tables. In practice, execution happens through `create_doc`, and possibly `create_sheet` if the user asks for a separate spreadsheet.
